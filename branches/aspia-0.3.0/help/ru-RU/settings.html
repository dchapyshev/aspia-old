Настройки программы

Aspia имеет ряд настраиваемых параметров, которые можно изменить перейдя в диалог настроек (рис. 1).

Рис. 1


В левой части диалога расположен список категорий параметров, который состоит из:

  • Основных параметров
  • Фильтра данных
  • Параметров системного трея


При нажатии кнопки «Сохранить» параметры программы будут сохранены и окно будет закрыто. При нажатии кнопки «Закрыть» окно будет закрыто без сохранения параметров.


Основные параметры

В категории «Основные» (рис. 1) пользователь может изменить язык приложения, используемую тему иконок, категорию, которая будет отображаться при запуске программы, а так же ряд следующих параметров:

  • Запускать вместе с Windows - параметр определяет будет ли Aspia автоматически запускаться при включении Windows.
  • Сохранять положение окон - будут ли сохраняться положения окон. Влияет на главное окно программы и диалог сохранения отчета. В главном окне программы при снятии этого флажка не будут сохраняться размеры колонок и положение разделителя между деревом категорий и открытой категорией.
  • Поверх всех окон - будет ли программа отображаться поверх всех окон.
  • Использовать стили для информационных окон - при наличии флажка будут использоваться стили оформления (рис. 2), а при отсутствии - соответственно не будут (рис. 3).

  • Чередующиеся строки в окне информации - при отображении информации будут использоваться чередующиеся темные и светлые строки, что должно улучшать восприятие. Если флажок не установлен, то все строки будут иметь белый фон.
  • Разрешить драйвер режима ядра - включение и отключение драйвера Aspia, который используется для получения данных SPD, DMI (SMBIOS) и информации с датчиков.
    При отключении драйвера часть функций программы станет недоступна.

Фильтр данных

Настройки в этой категории (рис. 4) позволяют выбрать наиболее важные данные при выводе информации и наоборот - отсеять те, которые пользователю не нужны.

Рис. 4

В настоящее время фильтрация данных доступна для категорий «Системные события» и «История IE». Параметры в данных настройках влияют и на вывод информации в окне программы, и при сохранении отчета.

Системный трей

Настройки в категории «Системный трей» (рис. 5) влияют на иконки, которые будут отображены программой в трее.

Рис. 5


  • Запускать свернутой в трей - при запуске программы она сразу же будет свернута в системный трей. При этом ее можно в любой момент восстановить нажав на иконку в трее левой кнопкой мыши. Если при установленном флажке «Запускать свернутой в трей» будет так же установлен флажок «Показывать иконку программы», то иконка программы будет отображаться при свернутом и развернутом главном окне. Если флажок «Показывать иконку программы» не будет установлен, то при восстановлении окна иконка будет удаляться из трея.
  • Показывать иконку программы - при установке этого флажка иконка Aspia всегда будет отображаться в системном трее (если программа запущена).
  • Показывать иконки датчиков - параметр включает отображение иконок датчиков в системном трее, которые разрешены в списке ниже.
  • Частота обновления - числовое значение, которое позволяет указать величину интервала (в секундах), с которым будет происходить обновление показаний на иконках датчиков в системном трее.
  • Иконки процессоров и иконки жестких дисков - кнопки в этих областях позволяют настроить цвет фона и цвет шрифта для иконок процессоров и жестких дисков. При нажатии этих кнопок выводится стандартный диалог выбора цвета.
